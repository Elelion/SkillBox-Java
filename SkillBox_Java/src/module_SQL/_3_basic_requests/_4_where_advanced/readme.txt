• LIKE - аналогичен знаку равенства, с возможностью добавлять в строку маски.
Те мы можем поставить букву с которой будет начинаться слово что мы ищем, а
заканчиваться оно будет как угодно.
% в данном случае обозначает любой символ до указанного нами слова или буквы
или после, смотря где он стоит: %word% / word% / %word, так же можно указывать
и пробелы % word % и тому подобное…

Например:
SELECT * FROM SkillBox_Shop.good_category WHERE `name` LIKE 'ч%';

• NULL - данное значение НЕ является 0, хотя результат может быть и одинаковым.
Null принято обозначать родительские поля. Например если написать IS NOT NULL,
то результат будет любой КРОМЕ NULL.
Для примера выполним предыдущий запрос и добавим к нему условие вывода
родительской категории.

Например:
SELECT * FROM SkillBox_Shop.good_category
WHERE `name` LIKE 'ч%'
AND `parent_id` IS NOT NULL;

• OR - или , позволяет задавать несколько условий. Те сделать условие1 ИЛИ
условие2 ИЛИ условие3
Например 1:
SELECT * FROM SkillBox_Shop.good_category
WHERE `name` LIKE '% чай %'
OR `name` LIKE 'чай %'
OR `name` LIKE '% чай';

Пример 2 (приоритет условию в скобках):
SELECT * FROM SkillBox_Shop.good_category
WHERE (`name` LIKE '% чай %'
OR `name` LIKE 'чай %'
OR `name` LIKE '% чай')
AND `parent_id` IS NOT NULL;

• IN - аналогично OR, является сокращенным для нескольких или условий как
и в случае с OR. Только применять LIKE с OR уже нельзя.

Например:
SELECT * FROM SkillBox_Shop.good_category
WHERE `name` IN ('чай', 'Ти', 'Мате', 'Кофе');
